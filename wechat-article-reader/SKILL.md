# 微信文章阅读器

## 核心能力

**必须保证的能力：**
- 能够提取和输出微信文章的完整内容（标题、作者、发布时间、摘要、正文）
- 保留所有原始信息（价格、数据、链接、细节），不做删改

## 快速开始

读取微信文章内容：

```bash
python3 scripts/read_wechat_article.py "https://mp.weixin.qq.com/s/xxxxxx"
```

脚本会输出文章的所有信息，后续根据用户需求灵活处理。

## 使用场景与输出策略

根据用户需求灵活处理，**核心是保证能获取完整内容**：

### 场景1：阅读完整内容
用户说："读一下这篇微信文章"、"文章链接是..."
→ 输出完整内容（标题、作者、发布时间、摘要、正文）

### 场景2：总结文章
用户说："总结这篇文章"、"这篇文章讲了什么"
→ 只输出总结，不输出完整内容

### 场景3：提取关键信息
用户说："提取关键信息"、"帮我找到价格"、"统计目的地"
→ 只输出提取的关键信息，不输出完整内容

### 场景4：分析或深入研究
用户说："分析一下"、"详细解读"
→ 可以基于完整内容做深入分析，但输出要聚焦在分析结果上

**输出原则：**
- 保证有能力获取完整内容（核心能力）
- 根据用户需求决定输出形式
- 不要同时输出完整内容和总结（避免冗余）

## 文章格式

微信文章URL格式：`https://mp.weixin.qq.com/s/xxxxxx`

脚本会自动提取以下信息：

- **标题** (title) - 文章主标题
- **作者** (author) - 公众号作者名称
- **发布时间** (publish_time) - 文章发布日期时间
- **摘要** (description) - 文章摘要描述
- **正文** (content) - 完整文章内容（保留段落结构）

## 工作流程

1. **验证URL** - 确认是微信文章链接（包含 `mp.weixin.qq.com`）
2. **发送请求** - 使用浏览器User-Agent模拟真实访问
3. **解析HTML** - 使用BeautifulSoup解析网页结构
4. **提取内容** - 提取标题、作者、时间、摘要和正文（保留所有原始信息）
5. **判断需求** - 根据用户请求决定输出形式：
   - 要求阅读 → 输出完整内容
   - 要求总结 → 只输出总结
   - 要求提取关键信息 → 只输出关键信息
6. **格式输出** - 按用户需求返回结果

**核心原则：**
- 必须有能力获取完整内容（价格、数据、链接、细节）
- 根据用户需求灵活输出，避免冗余
- 不要同时输出完整内容和总结

## 错误处理

如果文章无法访问，脚本会返回错误信息：

- **网络请求失败** - URL不可访问或网络问题
- **解析失败** - 页面结构异常
- **URL格式错误** - 不是微信文章链接
